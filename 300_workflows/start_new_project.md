# Workflow – Start New Project

**Version:** 0.1 Draft

## Purpose

Ein neues technisches Produktprojekt innerhalb des EOS starten.

## Steps

1. Projektziel beschreiben
2. Projektordner unter `800_projects/` anlegen
3. Project Charter erstellen
4. Betroffene Domänen identifizieren
5. Initiale Agenten bestimmen
6. Initiale Risiken erfassen
7. Initiale Requirements erfassen
8. Dokumentationsbedarf definieren
9. Review durch Orchestrator und Knowledge Manager
