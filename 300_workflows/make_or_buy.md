# Workflow – Make or Buy Decision

**Version:** 0.1 Draft

## Purpose

Strukturierte Make-or-Buy-Entscheidungen für Komponenten, Software, Tools oder Dienstleistungen.

## Required Inputs

- Requirement
- Stückzahl-/Business-Annahme
- Technische Kritikalität
- Zulassungsrelevanz
- Lieferantenoptionen
- Kostenannahmen
- Testaufwand

## Required Agents

- Engineering Orchestrator
- Chief Systems Engineer
- Fachagent der betroffenen Domäne
- Supplier Strategy Agent
- Cost Engineering Agent
- Compliance Agent, falls zulassungsrelevant
- Auditor, falls sicherheits- oder zulassungsrelevant

## Output

- ADR
- Risiken
- offene Punkte
- Empfehlung
