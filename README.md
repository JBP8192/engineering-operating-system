# Engineering Operating System (EOS)

**Status:** v0.1 Foundation Draft  
**Purpose:** Aufbau einer digitalen Engineering-Organisation aus Menschen, KI-Agenten, Prozessen, Werkzeugen und Wissensbeständen.

## Leitidee

Das Engineering Operating System existiert nicht nur, um technische Produkte zu entwickeln. Es existiert, um eine Organisation aufzubauen, die mit jedem Projekt besser darin wird, hervorragende technische Produkte zu entwickeln.

Jedes Projekt erzeugt daher zwei Ergebnisse:

1. **Produktwissen** – Wissen über das entwickelte technische System.
2. **Organisationswissen** – Wissen darüber, wie gute Entwicklung funktioniert.

## Repository-Struktur

```text
engineering-operating-system/
├── 000_constitution/          # Verfassung des EOS
├── 100_governance/            # Governance, Reviews, Freigaben, Change Management
├── 200_organization/          # Agentenorganisation, Rollen, Schnittstellen
├── 300_workflows/             # Standardprozesse und Arbeitsabläufe
├── 400_knowledge_model/       # Datenmodell: Requirements, Risiken, Tests, ADRs
├── 500_agent_specifications/  # Spezifikationen der Agenten
├── 600_tools/                 # OpenClaw, Telegram, GitHub, Confluence, MCPs
├── 700_templates/             # Vorlagen für ADRs, Requirements, Reviews usw.
├── 800_projects/              # Produktprojekte, z. B. Electric Scooter
└── 900_lessons_learned/       # Organisationswissen und Verbesserungen
```

## Grundprinzipien

- Systeme vor Komponenten
- Anforderungen vor Lösungen
- Wissen ist ein Produkt
- Entscheidungen müssen nachvollziehbar sein
- Qualität entsteht kontinuierlich
- Einfachheit besitzt Vorrang
- Unsicherheit wird transparent gemacht
- Zusammenarbeit ist verpflichtend
- Verbesserung endet niemals

## Erster Anwendungsfall
